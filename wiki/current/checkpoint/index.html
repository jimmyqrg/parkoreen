🏁 Checkpoint

Respawn markers with three visual states configurable in Map Config.

Overview

Checkpoints are flag/post objects the player walks through to save their progress. The most recently activated checkpoint is where the player respawns on death.

Visual states

StateDefault colorCondition
Inactive#808080 (gray)Never been touched by the player.
Touched#2196F3 (blue)Player passed it but it is not the current respawn point.
Active#4CAF50 (green)Current active respawn point.

All three colors are customizable in Map Config → Checkpoint Colors.

Respawn behavior

  • Only one checkpoint is active at a time.
  • Touching a later checkpoint deactivates earlier ones (they turn blue).
  • Re-touching a blue checkpoint makes it active again.
  • On death the player respawns centered on the active checkpoint.
  • If no checkpoint has been activated, the player respawns at the map's spawn point.
